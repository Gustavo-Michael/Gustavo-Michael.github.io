Actividad 04: Aplicación de iptables, Mecanismos de defensa en red

Objetivo: Implementación de políticas de seguridad restrictivas y permisivas en red utilizando reglas de firewall con iptables.

Ejercicios de Configuración

1. Establecer una política restrictiva iptables -A INPUT -i eth0 -p tcp -m multiport --dports 192.1.2.10, 192.1.2.11 -m state --state NEW, ESTABLISHED -j [DROP/REJECT] Nota: Se define el filtrado por puertos y estados de conexión.
2. Permitir el tráfico de conexiones ya establecidas iptables -A INPUT -p tcp --dport 192.1.2.10 -m state --state ESTABLISHED -j ACCEPT
3. Aceptar tráfico DNS (TCP) saliente de la red local iptables -A OUTPUT -p tcp --dport 53 -d 192.1.2.x -j ACCEPT Regla aplicada para permitir la resolución de nombres hacia el exterior.
4. Aceptar correo entrante proveniente de Internet iptables -A INPUT -p tcp --sport 25 -d 192.1.2.10 -j ACCEPT
5. Permitir correo saliente a Internet desde el servidor iptables -A OUTPUT -p tcp --dport 25 -s 192.1.2.10 -j ACCEPT
6. Aceptar conexiones HTTP desde Internet al servidor web iptables -A INPUT -p tcp --dport 80 -j ACCEPT
7. Permitir tráfico HTTP desde la red local a Internet iptables -A OUTPUT -p tcp --dport 80 -j ACCEPT

Conclusiones

La correcta configuración de iptables permite establecer un perímetro de seguridad robusto. Al combinar políticas por defecto restrictivas con reglas específicas para servicios críticos (como DNS, HTTP y SMTP), se minimiza la superficie de ataque disponible para posibles intrusos, garantizando la disponibilidad e integridad de los servicios de red[cite: 43, 44].
